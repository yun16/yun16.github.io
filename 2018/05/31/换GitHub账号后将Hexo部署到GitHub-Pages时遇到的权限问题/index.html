换GitHub账号后将Hexo部署到GitHub Pages时遇到的权限问题

发表于 2018-05-31 | 分类于 技术问题 |

出于各种原因注册了一个新的 GitHub 账户，然后想把 Hexo 部署到 GitHub Pages 上作为记录平常碎碎念的博客平台，整个过程网络上的资料很多，这里就不再复述，我主要参考了下面两篇文章的内容：

  1. 如何搭建一个独立博客——简明Github Pages与Hexo教程
  2. 使用Hexo+Github一步步搭建属于自己的博客（基础）
  3. GitHub + Hexo 搭建博客平台
  4. 让你的Hexo博客华丽升级
  5. 在Github上面搭建Hexo博客（四）:使用不同电脑维护

完成了大部分步骤之后，我在将 Hexo 部署到 GitHub Pages 时命令行报了错

remote: Permission to yun16/yun16.git denied to shuaiyunzhang.
fatal: unable to access 'https://github.com/yun16/yun16.git/': The requested URL returned error: 403


其中shuaiyunzhang是我前一个账号的名字，现在的是yun16，很显然这个是权限的问题，其间我花了很长时间去搜索报错的代码，在 GitHub 和 stack overflow 上看了很多相关问题，但没有找到类似情况的确切解决方案，后面转换搜索策略，对搜索关键词和搜索条件进行限制，终于找到了一篇文章Mac之Git账号的坑，里面提到了Mac多Git账号配置，顺藤摸瓜找到了 GitHub 官网的帮助文档

里面提到了两种方式：更新资质或者删除资质，我选择了第二种方式。

$ git credential-osxkeychain erase
host=github.com
protocol=https
[Press Return]

If it’s successful, nothing will print out. To test that it works, try and clone a repository from GitHub. If you are prompted for a password, the keychain entry was deleted.

然后执行hexo clean、hexo g、hexo d命令，成功部署。

题外话：个人觉得这个问题资料比较少的原因应该是很少有人会闲的去换自己的 GitHub 账号……
